  • Introduction

    This plan specifies the communication of the test appoach to "Tic Tac Toe Game". It includes all objectives, scope, test schedule and approach. This document will clearly ifentify what test deliverables will be.

    In modern society, we have a various of users and they have a various of devices. This test plan will also provide information about how well is the game supported over all the plantform.

    The whole testing will be mainly focus on three components, Unit testing, System Integration testing and Acceptance. The details of our test will be further defined in approach section and in level specific plans.

  • Test Items

    The test in this document will include three major components, including game board layout which will also includes basic game logic, check win functionality, button functionality which includes start new game function, save a game function, load a game function and exit the game.

  • Features To Be Tested

    Feature that will be tested include game board layout, game logic, check the win conditions, load a game, save a game, start a new game and exit a game.

    Also, pop up proper information when user clicks on certain buttons will also be tested.

    For a better reading expirience, we have extract and visualized all the features that need to be testd

  • Features Not To Be Tested

    Tic Tac Toe is a really small, yet complex system. However it does not consume much computing power, in this case, the judgement on its algorithm is not that important. There may be good algorithms than run faster than current one, but the result for end users is unrecongnizable. In this case, the algorithm will not be tested.

  • Approach

    For testing the whole system, we will be dividing testing into four major components, Unit testing, Intergration testing, System testing and acceptance testing.

    Arguments for tests will only be the mouse clicks. The whole system does not require any keyboard input or other input devices. So base on this argument, if we receive expected result, we will consider it is passed.

    It is expected that the appropiate symbol (cross or circle) will apear in the selected slot on the game board. When a player could get 3 of the symbols on a straigh line (horizontal, vertical or diangnal), there will be proper information displayed to the users. Save button is expected to save current game board and load buttong is expected to recover the saved game into current game board. In addition, any operation that will interept current game will arouse warning texts.

    To perform this test, there will be no additional dependencies. The test could be test on any devices that support browsers and javescript.
