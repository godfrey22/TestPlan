  • Introduction

    This plan specifies the communication of the test appoach to team members. It includes all objectives, scope, test schedule and approach. This document will clearly ifentify what test deliverables will be.

  • Test Items

    The test in this document will include three major components, including game board layout which will also includes basic game logic, check win functionality, button functionality which includes start new game function, save a game function, load a game function and exit the game.

  • Features To Be Tested

    Feature that will be tested include game board layout, game logic, check the win conditions, load a game, save a game, start a new game and exit a game.

    Also, pop up proper information when user clicks on certain buttons will also be tested.

  • Features Not To Be Tested

    Because this is a relatively small project and all the functionalities are absolutly essential to the whole user expirience, we will perform all the test items listed above, no exception.
